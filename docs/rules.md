# D&D 快速车卡规则约定

## 1. 文档目的

本文档是项目的规则基线，用于约束页面流程、数据模型、自动计算与合法性校验。第一阶段默认支持 **D&D 5e 2024 修订版**，不把 2014 版规则混入同一套角色数据。

## 2. 首期支持范围

- 规则集：`5e-2024`
- 等级：1 级
- 内容来源：公开免费规则和可合法使用的 SRD 内容
- 属性生成：标准数组、27 点购点、自定义录入
- 使用方式：手机端优先，同时兼容桌面浏览器
- 数据保存：优先支持浏览器本地保存与 JSON 导入、导出
- 暂不包含：多职业、高等级升级、完整商业扩展书内容、DM 房规编辑器

## 3. 开卡前的团规则

创建角色之前，需要确定：

1. 使用的规则版本；
2. 起始等级；
3. 允许使用的规则来源；
4. 属性生成方式；
5. 是否允许专长、多职业或自定义背景；
6. 初始装备或金币规则；
7. DM 禁用、替换或新增的内容。

第一版将这些条件固化为 5e 2024 的默认配置，后续再扩展为可分享的“团规则”。

## 4. 2024 版车卡流程

### 4.1 选择职业

职业决定生命骰、主要属性、豁免熟练、护甲与武器熟练、技能选项、1 级职业能力以及施法方式。

完成条件：职业必选项全部完成，职业给予的熟练与能力已写入角色状态。

### 4.2 确定角色起源

角色起源由出身、物种和语言组成。

出身通常提供：

- 三项允许提升的属性；
- 一项 `+2` 与另一项 `+1`，或三项各 `+1`；
- 一个起源专长；
- 两项技能熟练；
- 一项工具熟练；
- 装备包或对应金币选项。

物种提供体型、速度、感官、抗性、天生能力或天生法术等特性。2024 版中，属性提升来自出身，不来自物种。

完成条件：出身、物种、语言及其所有派生选择均已完成，且没有非法重复选择。

### 4.3 生成六项属性

六项属性为：力量 STR、敏捷 DEX、体质 CON、智力 INT、感知 WIS、魅力 CHA。

属性调整值公式：

```text
modifier = floor((score - 10) / 2)
```

完成条件：六项属性都有合法数值，生成方式满足当前团规则，出身加值分配合法，最终属性没有超过规则上限。

### 4.4 选择阵营与人物细节

填写名字、阵营、外貌、性格、理想、羁绊、缺点和背景故事。阵营是人物倾向的描述，不作为限制玩家行为的硬性程序规则。

完成条件：名字等必填字段完整；叙事字段允许稍后补充。

### 4.5 选择装备

装备需要区分“拥有”和“已装备”。只有已穿戴、已持握或已同调且满足条件的物品，才能影响 AC、攻击或其他派生数值。

完成条件：职业与出身的装备选择完整，不存在互斥选择冲突。

### 4.6 配置法术

施法角色需要记录施法属性、戏法、已知法术或准备法术、法术位及特殊来源法术。必须区分：会使用、已准备、拥有法术位和每日免费使用次数。

完成条件：法术数量、等级、职业列表与前置条件均合法。

### 4.7 最终校验

系统至少检查：

- 必选项是否遗漏；
- 属性与出身加值是否合法；
- 技能、工具与语言选择数量是否正确；
- 重复熟练是否被错误累计；
- 装备选择是否完整；
- 法术数量与来源是否合法；
- HP、AC、先攻、攻击加值和法术 DC 是否计算正确。

## 5. 核心计算规则

### 5.1 熟练加值

1 级角色的熟练加值为 `+2`。熟练加值只能在规则明确允许时加入，普通熟练通常不能因多个来源重复累计。

### 5.2 技能与豁免

```text
技能加值 = 对应属性调整值 + 熟练加值（若熟练）+ 其他修正
豁免加值 = 对应属性调整值 + 熟练加值（若熟练）+ 其他修正
```

专精通常将该项使用的熟练加值变为两倍，而不是添加一次新的普通熟练。

### 5.3 生命值

```text
1级最大生命值 = 职业生命骰最大值 + 体质调整值 + 其他修正
```

### 5.4 护甲等级

未使用其他 AC 公式时：

```text
AC = 10 + 敏捷调整值
```

护甲、天生护甲和无甲防御通常提供不同的 AC 计算公式，应选择一个适用公式，再加入盾牌或规则明确允许叠加的修正，不能把多个基础公式直接相加。

### 5.5 攻击与伤害

```text
攻击加值 = 相关属性调整值 + 熟练加值（若熟练）+ 其他修正
伤害 = 武器伤害骰 + 相关属性调整值 + 其他修正
```

熟练加值通常不加入伤害。

### 5.6 法术

```text
法术攻击加值 = 熟练加值 + 施法属性调整值 + 其他修正
法术豁免 DC = 8 + 熟练加值 + 施法属性调整值 + 其他修正
```

### 5.7 先攻与被动察觉

```text
先攻 = 敏捷调整值 + 其他修正
被动察觉 = 10 + 察觉技能总加值 + 其他修正
```

## 6. 产品数据原则

- 每条规则数据必须带 `ruleset`，例如 `5e-2024`。
- 规则数据、角色选择和派生数值分开保存。
- 派生数值通过规则函数计算，不作为唯一事实源手工维护。
- 每项熟练、能力、法术和修正保留来源，便于解释“为什么得到这个数值”。
- UI 负责展示和收集选择，不在 Vue 组件中硬编码职业规则。
- 非法选项优先给出原因；只有规则明确禁止时才阻止继续。

## 7. MVP 验收标准

一个不了解规则的新玩家应能只依靠页面提示，在约 5 至 10 分钟内创建一张规则合法的 1 级角色卡，并能在手机上快速找到：

- AC、HP、先攻和速度；
- 六项属性与豁免；
- 技能；
- 攻击与伤害；
- 职业、出身和物种能力；
- 法术与可用资源。
